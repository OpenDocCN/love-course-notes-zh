# 公文写作法：02：如何提出让老板眼前一亮的建议？（上）

在本节课中，我们将学习商业建议书（Proposal）的写作方法。我们将探讨为何许多建议难以被采纳，并重点讲解建议书中“背景分析”部分的构建逻辑与技巧。

上一节我们介绍了其他类型的公文写作，本节中我们来看看最后一种常用文体——商业建议书。

相信很多同学都有过这样的经历。在公司工作一段时间之后，我们发掘了一些工作中可以被优化或者改进的部分，于是兴冲冲地向自己的老板或上级提出建议。满以为自己的提议能够帮老板解决燃眉之急，可没成想建议非但没有引起任何反响，还很快被老板抛到脑后，让我们的内心备受打击。

实际上，站在老板的角度，并不是不欢迎大家提建议。公司要向前发展，光靠老板一个人牵引是远远不够的，也需要整个团队的助推。因此，靠谱的商业建议对于老板和公司都很有价值。我们也一直鼓励大家用老板的思维来看待工作中遇到的问题。

但是，究竟是什么原因导致我们的商业建议总是难以被采纳和接受？多半是因为我们用错了方法。

在提出商业建议时，有几个比较常见且严重的误区，需要格外注意。

以下是几个常见的误区：

1.  **过于理想化，不接地气**。许多高屋建瓴的建议，在老板看来会觉得挺幼稚。比如下一步要扩充哪个业务线、公司组织架构如何调整等。不是说这些建议本身不好，而是它们多半已被老板或高层想到过，又因种种原因被否决，或在实践中被证明不可行。对于老板而言，再去听取一遍已被否定的建议，无异于浪费宝贵的时间。
2.  **逻辑结构混乱**。许多建议在提出时并没有做过详细的逻辑梳理。例如，我们常常看到一些建议是这样呈现的：“针对当前存在的效率低下问题，我提议加强内部流程制度管理，提高生产效率，妥善运用工作时间，缩短业务线。”在这样的汇报中，老板能接收到的信息是散点状的，不够连贯和体系化，很难从中得到下一步的操作方案。即便每一条建议都正确，也会因为结构混乱而难以发挥应有作用。
3.  **缺乏数据模型支撑**。很多建议之所以难以落地，很大一部分原因是没有给出可借鉴的财务模型。例如，要投入公司多少人力、物力和财力？投入的性价比如何？这会导致老板在决策时，由于不清楚建议会带来何种成本与资源投入而产生怀疑，从而出于保险考虑，最终决定不予采纳。

老板们的时间和精力是非常稀缺的资源，而我们为商业建议所付出的时间也同样宝贵。如果希望自己提出的建议能够发挥最大效用，那么接下去的内容就需要我们留心听讲。

首先，请大家思考一下，商业建议类的单页文档（One Pager），首先应该出现的内容是什么？

可能有的同学会说，当然是要说明我们的建议是什么，能够解决什么问题。这种思路听上去非常顺理成章。但实际上，在这之前还少了非常重要的一步，那就是有关背景的分析和介绍。

背景分析介绍的作用主要有两个：一是让读者明白这个建议所处的宏观环境是怎样的，是在怎样的条件下应运而生的；二是明确这个建议核心的利益点，到底解决了什么问题。

所以，这一部分的分析通常需要分为以下几个维度。

以下是背景分析的几个关键维度：

1.  **外部环境数据**。这一部分要分为静态和动态两个方面去做。
    *   **静态**：指从宏观角度介绍行业整体状况。例如，`电商行业的平均毛利率`、`短视频行业的整体用户体量`、`保险行业的整体市场规模`等，把当前的市场状况客观呈现出来。
    *   **动态**：指我们需要通过对数据的分析和解读，得出当前市场的发展走向，并由此引出下面要提出的建议。例如：“微信公众号数量超过2000万个，增幅有所放缓，而短视频和小程序呈现出新的流量洼地。其中，**首X活跃用户数2.28亿人，环比增长0.6%，位居榜首；排名第二的是抖音短视频，活跃用户数为1.92亿人，环比增长5.6%**。”读到这里，读者自然会明白，我们下面要提出的建议可能与短视频有关。
2.  **公司内部的经营情况以及主要的经营数据分析**。分析完外部环境数据之后，就要来反观我们自身的发展情况，同样分为静态和动态两个方面。
    *   **静态**：需要先客观分析公司当前的经营状况。例如，每个板块的收入、利润情况，各自对收入的贡献率如何。
    *   **动态**：则需要能够说明一定趋势的比率来加入分析。例如：“**电商利润率53%，收入200万，同比上涨20%**。**广告收入50万，同比下降15%**。”类似这样的表述，能够很快让读者明确当前各项业务所处的趋势。这样，如果后面我们提出了顺应这种趋势的建议，读者的理解成本也会相对降低。
3.  **前瞻性或结论性表述**。在做完以上两部分的分析之后，背景分析这里还需要一项内容，那就是一个概括性或结论性的叙述来进一步印证我们的观点。例如：“**短视频行业的增长潜力增大，预计明年市场规模能够达到500亿**。”

有关背景的介绍内容讲到这里，读者基本上能够从中读到一个全景性的信息，知道我们即将要给出的建议是基于怎样的事实背景而产生的，能够减少他们在后面阅读过程中的很多疑问。

与此同时，还有一个小技巧可以教给大家。由于背景分析大多是针对已经发生过的事情，无论是比率、数字还是趋势，因此，如果觉得单纯列举数据不足以说明问题，还可以借助图表。

一方面让文档显得更加专业和美观，另一方面也让我们的分析更直观地呈现给读者。

以下是几种常用的图表及其作用：

*   **柱状图**：通过不同高度的柱形来呈现不同阶段或不同项目的表现情况，侧重横向对比。
*   **折线图**：作用类似柱状图，但更加侧重趋势的分析，通过散点的连接来揭示某种趋势，当然也可用作对比。
*   **饼图**：侧重于比例的表示，通过对一个圆形的分割来表示不同类型的项目在整体中所占的比重。

打个比方，当我们要回顾近三年的业绩增长情况，来说明当前的业务量正在扩张或缩水时，可以横向对比营业收入、利润、成本这三个维度的数据，并在此基础上叠加利润率的折线图来将三者整合起来。这样我们就可以清楚地看到整体的营收和利润变动情况了。

有关Proposal中背景介绍的部分，我们就先介绍这么多。下节课我们将继续为大家讲解关于Proposal的后面两个部分，也就是方案分析和财务模型分析的内容。

本节课中，我们一起学习了商业建议书写作中“背景分析”部分的重要性与构建方法。我们首先分析了建议不被采纳的常见误区，然后详细讲解了如何从外部环境、内部经营数据和前瞻性结论三个维度，并辅以图表，来搭建一个坚实、有说服力的建议背景，为后续提出具体方案奠定基础。
