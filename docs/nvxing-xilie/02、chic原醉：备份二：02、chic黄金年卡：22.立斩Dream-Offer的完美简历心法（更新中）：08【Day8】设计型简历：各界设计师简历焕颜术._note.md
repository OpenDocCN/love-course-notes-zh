# 简历整容术：02：设计型简历焕颜术

在本节课中，我们将学习如何为设计师岗位打造一份既专业又高效的简历。我们将从招聘方（HR与设计部门负责人）的双重视角出发，解析他们的核心诉求，并提供具体的简历优化策略。

## 课程概述

设计师岗位涵盖产品设计师、游戏设计师、UI设计师、建筑设计师、服装设计师等多个领域。我们将这类简历统称为设计型简历。你可能会认为，设计师理应擅长设计一份美观的简历。然而，实际情况是，设计师的求职简历常常是让招聘者感到棘手的类型之一。

## 招聘方的双重审阅流程

在深入细节之前，我们首先需要了解企业筛选设计师简历的典型流程。针对设计师这类强技能、强思维的岗位，简历筛选通常分为两步：首先由HR进行初步筛选，随后由设计部门负责人进行专业评估。

HR会依据岗位基本要求进行快速筛选，例如考察**设计领域**、**毕业院校**以及**曾服务企业的级别**。通过初筛的简历会被转交给设计部门负责人，后者将重点考察应聘者的**设计思维**与**设计能力**，例如其参与过的**设计项目**和**设计作品**。只有双方都满意，应聘者才会获得面试邀请。

你会发现，HR与设计部门负责人对简历的关注点并不相同。接下来，我们将分别探讨他们的核心诉求。

## HR的诉求：信息提取的便捷性

上一节我们介绍了招聘的双重流程，本节中我们来看看HR的具体诉求。与许多设计师的设想不同，HR并非极度关注简历的“设计感”，他们的核心诉求在于**信息提取的便捷度**。例如，他们需要快速确认你是否毕业于美院，是否有4A广告公司经历等。

因此，许多设计师耗费大量精力设计的创意模板或花式排版，其实际效用可能低于预期。例如，近期流行用**技能条**或**百分比图表**来展示技能水平（如 `Photoshop: 85%`）。虽然视觉效果不错，但HR普遍认为其参考价值有限，因为“85%”所对应的具体操作水平并不明确。

这类简历如同时装秀上的服装，代表了设计理念，但缺乏日常穿着的实用性。简历设计的根本目的是提高投递成功率，一切设计都应围绕**使用的便捷性**展开。

以下是针对HR便捷性诉求的三点设计建议：

1.  **在HR审阅体验地图上融入创意**：遵循简历使用体验优先的原则。例如，一位漫画师将自己的职业历程绘制成黑白风格的故事漫画，按照HR的阅读顺序讲述亮点，既突出了专业能力，又增添了趣味性。
2.  **尽量使用Word格式**：许多设计师喜欢将简历导出为JPG等图片格式，但这不利于HR复制粘贴你的个人信息（用于制作面试安排表等）。为了提供更便捷的体验，建议优先使用可编辑的Word格式。
3.  **控制作品集文件大小**：确保作品集压缩包的大小便于HR下载和转发。建议将文件大小控制在几十兆以内，**20MB为最佳**，因为这是许多即时通讯工具（如微信）的单文件传输上限。你可以通过**降低分辨率**或使用**JPEG格式**（品质设为最大值）进行压缩。

关于如何满足HR对信息便捷提取的需求，我们就讨论到这里。接下来，我们将视角转向设计部门负责人。

## 设计部门负责人的诉求：彰显思维与能力

课程开头时我们提到，设计部门负责人主要关注应聘者的**设计思维**和**设计能力**。他们通常通过**设计岗位工作经历**和**设计作品集**来进行初步判断。

### 如何撰写工作经历

首先，我们来看看如何在工作经历描述中突出个人能力。许多设计师的简历描述流于泛泛，例如：“负责某某设计项目，分析业内流行趋势，跟进项目持续优化产品体验。” 这类描述缺乏个性，辨识度低。

要提高辨识度并在工作经历中突出设计能力，你可以使用以下撰写公式：

**个人行为 + 定向内容 + 操作方法 + 设计反馈与数据结果**

我们以产品设计岗位为例进行拆解：
*   **个人行为**：负责产品视觉设计。
*   **定向内容**：确立视觉风格规范及视觉元素。
*   **操作方法**：通过对竞品分析、设计趋势和用户买点数据进行分析，重构旧版视觉。
*   **设计反馈与数据结果**：新版设计上线运营4个月，新增25万用户，70%的用户反馈提及新版视觉体验有明显提升。

在公式的加持下，个人设计能力得到了更具说服力的展示，特别是**定向内容**、**操作方法**和**设计反馈**，构成了你独一无二的个性化内容。

### 如何打造会“说话”的作品集

分享完工作经历的撰写方法，我们再来说说如何通过作品集彰显个人设计思维。对于设计师而言，准备作品集是必要的，但关键在于制作一份能传达设计思维的、会“说话”的作品集。

以下是需要注意的几点：

首先，作品集应具有**层次感**。它无需包含所有作品，而应按照作品或项目类型进行精选。例如，可以按**活动海报**、**品牌手册**、**热点营销**等分类，具体类型需根据应聘岗位涉及的设计种类而定。每类下展示两三个精品即可，并按照与岗位的相关度进行排序，将最重要的作品前置。此外，建议在作品集最后添加**业余作品**（如二次元创作、手绘、临摹等），这能为负责人留下更立体的能力画像。

其次，作品集不仅用于展示，更有重要的**说明功能**。因此，需要在作品中融入关键信息，例如：
*   设计主题与要求
*   设计思路与亮点
*   作品寓意
*   项目耗时
*   所用工具

这些说明信息能帮助设计负责人更清晰地感知你的设计思维，加速其对作品的鉴赏和个人能力的评估。这一点睛之笔常被设计师忽视。

## 课程总结

本节课中，我们一起学习了如何针对设计师岗位优化简历。核心知识点总结如下：

1.  **双重审阅**：设计师简历需经过HR和设计部门负责人的双重审阅，两者关注点不同。
2.  **HR的核心诉求是便捷性**：为满足HR，建议在保证信息提取便捷的基础上增加创意，使用Word格式简历，并将作品集大小控制在20MB以内。
3.  **设计部门负责人的核心诉求是能力展示**：为满足专业评审，建议使用 **`行为 + 定向内容 + 操作方法 + 反馈与结果`** 的公式撰写工作经历，并在作品集中添加详细的设计说明，以突出设计思维与个性化能力。
