## 核心概念解析

上一节我们介绍了网络模型的基础，本节中我们来看看几个容易混淆的核心术语。

**1. 权重初始化**
权重初始化对模型训练至关重要。不恰当的初始化可能导致梯度消失或爆炸。常用的方法是Xavier初始化，其公式为：
`W = np.random.randn(fan_in, fan_out) / np.sqrt(fan_in)`

**2. 学习率衰减**
固定学习率可能在训练后期阻碍收敛。学习率衰减策略能动态调整步伐，例如指数衰减公式为：
`new_learning_rate = initial_learning_rate * decay_rate ^ (step / decay_steps)`

**3. 梯度裁剪**
在训练深度网络时，梯度可能变得非常大（爆炸）。梯度裁剪通过设定阈值来限制梯度范围，代码示例如下：
```python
grad_norm = np.linalg.norm(gradients)
if grad_norm > max_norm:
    gradients = gradients * max_norm / grad_norm
```

## 常见操作问题与解决方案

理解了核心概念后，我们来看看实际操作中可能遇到的困难。以下是几个典型问题及其解决方法。

**问题一：训练损失不下降**
*   **可能原因**：学习率设置不当、模型架构存在缺陷、数据未正确预处理。
*   **解决方案**：
    *   尝试降低或增加学习率。
    *   检查网络层设计，确保信息能有效流动。
    *   验证输入数据是否已标准化或归一化。

**问题二：模型过拟合**
*   **可能原因**：模型过于复杂、训练数据不足、缺乏正则化。
*   **解决方案**：
    *   引入Dropout层，代码如 `tf.keras.layers.Dropout(0.5)`。
    *   使用L1或L2权重正则化。
    *   增加训练数据量或使用数据增强技术。

**问题三：评估指标波动大**
*   **可能原因**：验证集划分不合理、批次大小（Batch Size）设置过小。
*   **解决方案**：
    *   确保验证集数据分布与训练集一致。
    *   适当增大批次大小，以获得更稳定的梯度估计。

## 调试与优化技巧

掌握了问题解决方法，我们还需要一些工具和技巧来辅助调试。以下是一些实用的建议。

**技巧一：使用TensorBoard可视化**
TensorBoard可以帮助你直观地监控损失曲线、权重分布等，是调试的利器。

**技巧二：模块化测试**
不要一次性构建完整模型。应先确保每个独立模块（如单个层、损失函数）能正确运行。

**技巧三：简化问题**
遇到复杂问题时，尝试在一个极简的数据集或模型上复现并解决，再迁移到原任务中。
